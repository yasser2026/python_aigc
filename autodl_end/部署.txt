# AutoDL GPU 端部署 — 生成动画 + 多角色声音
#
# 适用：2×RTX 4090（24G）或同等 2×24G 双卡机器
# 编排服务（FastAPI :8091）跑在本机 Windows / 云 CPU，AutoDL 只跑 GPU 推理
#
# 架构：
#   本机 8091（python run.py）
#     ├─ COMFYUI_ANIME_HOST  → AutoDL GPU0 :8188  出图 + Wan I2V
#     └─ GPTSOVITS_HOST      → AutoDL GPU1 :9880  多角色声纹克隆
#
# 双卡分工（推荐）：
#   GPU0 → ComfyUI（显存占用大，动画/出图串行换工作流）
#   GPU1 → GPT-SoVITS 常驻（TTS 与出图可并行，对白多的集更快）

================================================================================
零、租机与镜像
================================================================================

1. 实例
   - GPU：2 × RTX 4090 24G（或 2×24G）
   - 数据盘：建议 50～80G（I2V-14B-480P + 出图 + TTS；后续换 720P 建议 80～100G）
   - 镜像：PyTorch 2.4/2.5 + Python 3.10/3.11 + CUDA 12.x

2. 开机后检查
   nvidia-smi
   # 应看到 2 张卡，驱动/CUDA 正常

3. 目录约定（AutoDL 上）
mkdir  /root/autodl-tmp/comfyui
mkdir  /root/autodl-tmp/models  
mkdir /root/autodl-tmp/gpt-sovits
mkdir  /root/autodl-tmp/voice_refs

   /root/autodl-tmp/models       # 大模型放数据盘（软链到 ComfyUI）

   /root/autodl-tmp/voice_refs   # 各角色参考音（与本机 data_anime 同步）

二、GPT-SoVITS（GPU1，端口 9880）
================================================================================

1. 安装（以官方 GPT-SoVITS v2 api 为例）
   cd /root/autodl-tmp
   # git 慢可本机下 ZIP 上传解压为 gpt-sovits/
   git clone https://github.com/RVC-Boss/GPT-SoVITS.git gpt-sovits
   cd gpt-sovits
   pip install -r requirements.txt
   pip install soundfile fastapi uvicorn ffmpeg-python   # api_v2 常缺依赖
   # 按官方文档下载预训练权重到 GPT_SoVITS/pretrained_models

2. 启动 API（绑定 GPU1；单卡实例改 CUDA_VISIBLE_DEVICES=0）

   ── 前台启动（调试推荐，日志直接打在终端，Ctrl+C 停止）──
   cd /root/autodl-tmp/gpt-sovits
   CUDA_VISIBLE_DEVICES=1 python api_v2.py -a 0.0.0.0 -p 9880

   ── 后台启动（长期挂着，日志写文件）──
   cd /root/autodl-tmp/gpt-sovits
   mkdir -p /root/autodl-tmp/logs
   CUDA_VISIBLE_DEVICES=1 nohup python api_v2.py -a 0.0.0.0 -p 9880 \
     > /root/autodl-tmp/logs/gptsovits.log 2>&1 &

   # 若 api 入口脚本名不同，以仓库 README 为准（常见为 api.py / api_v2.py）
   # 后台启动失败看日志：cat /root/autodl-tmp/logs/gptsovits.log

3. 验证
   curl http://127.0.0.1:9880/
   # 返回非 5xx 即服务在线

4. 多角色声音原理
   - 本机 pipeline 按 scene.narration_speaker_id 选角色
   - 每个角色需要参考音频：
     data_anime/{小说名}/characters/{角色id}/voice_ref.wav
   - 可选 voice_ref_text（与参考音一致的文本，克隆更准）

5. 参考音同步到 AutoDL（本机与 GPU 不共享磁盘时必做）
   在本机 config/gptsovits.json 增加（或 .env 同路径配置）：
     "remote_ref_audio_dir": "/root/autodl-tmp/voice_refs/{小说名}/characters"
   同步示例（本机 PowerShell / scp，替换为 AutoDL SSH 信息）：
     scp -P <端口> data_anime/盘龙/characters/char_001/voice_ref.wav \
       root@<autodl-ssh>:/root/autodl-tmp/voice_refs/盘龙/characters/char_001/voice_ref.wav
   目录结构需与 character_id 一致。

五、流水线阶段与 GPU 占用（便于排错）
================================================================================

   ─────────────────────────────────────────
   generate_images   ComfyUI 出图          GPU0

六、开机一键脚本（AutoDL 上，可选）
================================================================================

创建 /root/autodl-tmp/start_gpu.sh：

   #!/bin/bash
   set -e
   mkdir -p /root/autodl-tmp/logs

   # ComfyUI @ GPU0
   cd /root/autodl-tmp/comfyui
   CUDA_VISIBLE_DEVICES=0 nohup python main.py --listen 0.0.0.0 --port 8188 \
     >> /root/autodl-tmp/logs/comfyui.log 2>&1 &
   echo "ComfyUI PID $!"

   # GPT-SoVITS @ GPU1
   cd /root/autodl-tmp/gpt-sovits
   CUDA_VISIBLE_DEVICES=1 nohup python api_v2.py -a 0.0.0.0 -p 9880 \
     >> /root/autodl-tmp/logs/gptsovits.log 2>&1 &
   echo "GPT-SoVITS PID $!"

   sleep 3
   curl -s http://127.0.0.1:8188/system_stats | head -c 200; echo
   curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:9880/; echo

   chmod +x /root/autodl-tmp/start_gpu.sh

关机前在 AutoDL 控制台「关机」；不用时关机，按量计费。

八、常见问题
================================================================================

1. ECONNREFUSED / proxy error
   → 本机 run.py 未启动，或 .env 里 AutoDL 地址/端口填错

2. ComfyUI 有响应但生成失败
   → 工作流节点 id 与 config input_mappings 不一致；或模型未下载完整

3. 多角色声音都一样 / 变成 Edge 音色
   → 缺少 voice_ref.wav；或 GPT-SoVITS 不可达触发 fallback_to_edge
   → 检查 data_anime/.../voice_ref.wav 与 remote_ref_audio_dir 同步

4. OOM 显存不足
   → 减少 wan_i2v 单镜时长 / 帧数；或 ComfyUI 工作流里降 steps
   → 确保 ComfyUI 只在 GPU0，GPT-SoVITS 只在 GPU1

5. 升级 720P 高清动画
   → 下 Wan-AI/Wan2.1-I2V-14B-720P，改 config/wan_i2v.json high 档 model 与分辨率，default_tier 可改 high

6. 对白镜没有口型
   → 本方案不部署 InfiniteTalk；对白镜走 Wan I2V + TTS，属预期行为
   → 若需对口型，需另装 InfiniteTalk 节点/权重并改 pipeline.anime.json talk.enabled

7. 仅「生成视频」、不需要 GPU
   → video 模式用 Qwen API + Edge TTS，本机 CPU 即可，无需 AutoDL
